## List all websites

<CodeGroup>
```bash cURL
curl -X GET https://analytics.example.com/api/websites \
  -H "Authorization: Bearer spk_selfhosted_abc123..."
```
</CodeGroup>

<ResponseField name="data" type="array">
  Array of website objects
  <Expandable title="Website object">
    <ResponseField name="id" type="string" required>
      Website ID (format: `site_` + 10 alphanumeric chars)
    </ResponseField>
    <ResponseField name="tenant_id" type="string | null">
      Tenant ID (always `null` in self-hosted mode)
    </ResponseField>
    <ResponseField name="name" type="string" required>
      Website display name
    </ResponseField>
    <ResponseField name="domain" type="string" required>
      Website domain (normalized, without scheme/port)
    </ResponseField>
    <ResponseField name="timezone" type="string" required>
      IANA timezone (e.g., `America/New_York`)
    </ResponseField>
    <ResponseField name="share_id" type="string | null">
      Public share ID if sharing is enabled
    </ResponseField>
    <ResponseField name="created_at" type="string" required>
      ISO 8601 timestamp
    </ResponseField>
    <ResponseField name="updated_at" type="string" required>
      ISO 8601 timestamp
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pagination" type="object">
  <Expandable title="Pagination object">
    <ResponseField name="total" type="integer" required>
      Total number of websites
    </ResponseField>
    <ResponseField name="limit" type="integer" required>
      Items per page (1-100, default 20)
    </ResponseField>
    <ResponseField name="cursor" type="string | null">
      Next page cursor (website ID to continue from)
    </ResponseField>
    <ResponseField name="has_more" type="boolean" required>
      Whether more pages exist
    </ResponseField>
  </Expandable>
</ResponseField>

### Query Parameters

<ParamField query="limit" type="integer" default="20">
  Number of websites to return (1-100)
</ParamField>

<ParamField query="cursor" type="string">
  Website ID to paginate from (from previous response's `pagination.cursor`)
</ParamField>

### Response Example

```json
{
  "data": [
    {
      "id": "site_a1b2c3d4e5",
      "tenant_id": null,
      "name": "My Blog",
      "domain": "example.com",
      "timezone": "UTC",
      "share_id": null,
      "created_at": "2026-03-01T10:00:00Z",
      "updated_at": "2026-03-01T10:00:00Z"
    }
  ],
  "pagination": {
    "total": 1,
    "limit": 20,
    "cursor": null,
    "has_more": false
  }
}
```

## Create a website

<CodeGroup>
```bash cURL
curl -X POST https://analytics.example.com/api/websites \
  -H "Authorization: Bearer spk_selfhosted_abc123..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My Blog",
    "domain": "example.com",
    "timezone": "America/New_York"
  }'
```
</CodeGroup>

### Request Body

<ParamField body="name" type="string" required>
  Website display name (must not be empty)
</ParamField>

<ParamField body="domain" type="string" required>
  Website domain. Must be:
  - A valid hostname without scheme, port, path, or credentials
  - A registrable domain (not a public suffix like `.com`)
  - Lowercase, normalized automatically
  
  Valid: `example.com`, `blog.example.com`, `localhost`, `192.168.1.1`  
  Invalid: `https://example.com`, `example.com:3000`, `com`, `co.uk`
</ParamField>

<ParamField body="timezone" type="string" default="UTC">
  IANA timezone identifier (e.g., `America/New_York`, `Europe/London`)
</ParamField>

### Response (201 Created)

<ResponseField name="data" type="object">
  <Expandable title="Created website">
    <ResponseField name="id" type="string" required>
      Website ID (format: `site_` + 10 alphanumeric chars)
    </ResponseField>
    <ResponseField name="tenant_id" type="string | null">
      Always `null` in self-hosted mode
    </ResponseField>
    <ResponseField name="name" type="string" required>
      Website display name
    </ResponseField>
    <ResponseField name="domain" type="string" required>
      Normalized domain
    </ResponseField>
    <ResponseField name="timezone" type="string" required>
      IANA timezone
    </ResponseField>
    <ResponseField name="tracking_snippet" type="string" required>
      Ready-to-use HTML tracking snippet
    </ResponseField>
    <ResponseField name="created_at" type="string" required>
      ISO 8601 timestamp
    </ResponseField>
  </Expandable>
</ResponseField>

### Response Example

```json
{
  "data": {
    "id": "site_a1b2c3d4e5",
    "tenant_id": null,
    "name": "My Blog",
    "domain": "example.com",
    "timezone": "America/New_York",
    "tracking_snippet": "<script defer src=\"https://analytics.example.com/s.js\" data-website-id=\"site_a1b2c3d4e5\"></script>",
    "created_at": "2026-03-01T10:00:00Z"
  }
}
```

## Get a website

<CodeGroup>
```bash cURL
curl -X GET https://analytics.example.com/api/websites/site_a1b2c3d4e5 \
  -H "Authorization: Bearer spk_selfhosted_abc123..."
```
</CodeGroup>

### Path Parameters

<ParamField path="id" type="string" required>
  Website ID (format: `site_*`)
</ParamField>

### Response (200 OK)

<ResponseField name="data" type="object">
  Website object (same structure as in list endpoint)
</ResponseField>

### Response Example

```json
{
  "data": {
    "id": "site_a1b2c3d4e5",
    "tenant_id": null,
    "name": "My Blog",
    "domain": "example.com",
    "timezone": "America/New_York",
    "share_id": null,
    "created_at": "2026-03-01T10:00:00Z",
    "updated_at": "2026-03-01T10:00:00Z"
  }
}
```

### Error Responses

<ResponseField name="404 Not Found">
  ```json
  {
    "error": {
      "code": "not_found",
      "message": "Website not found"
    }
  }
  ```
</ResponseField>

## Update a website

<CodeGroup>
```bash cURL
curl -X PUT https://analytics.example.com/api/websites/site_a1b2c3d4e5 \
  -H "Authorization: Bearer spk_selfhosted_abc123..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My Updated Blog",
    "timezone": "UTC"
  }'
```
</CodeGroup>

### Path Parameters

<ParamField path="id" type="string" required>
  Website ID (format: `site_*`)
</ParamField>

### Request Body

All fields are optional. Only provided fields will be updated.

<ParamField body="name" type="string">
  New website display name
</ParamField>

<ParamField body="domain" type="string">
  New domain (must pass same validation as create endpoint)
</ParamField>

<ParamField body="timezone" type="string">
  New IANA timezone identifier
</ParamField>

### Response (200 OK)

<ResponseField name="data" type="object">
  <Expandable title="Updated website">
    <ResponseField name="id" type="string" required>
      Website ID
    </ResponseField>
    <ResponseField name="name" type="string" required>
      Updated name
    </ResponseField>
    <ResponseField name="domain" type="string" required>
      Updated domain
    </ResponseField>
    <ResponseField name="timezone" type="string" required>
      Updated timezone
    </ResponseField>
    <ResponseField name="updated_at" type="string" required>
      ISO 8601 timestamp of update
    </ResponseField>
  </Expandable>
</ResponseField>

### Response Example

```json
{
  "data": {
    "id": "site_a1b2c3d4e5",
    "name": "My Updated Blog",
    "domain": "example.com",
    "timezone": "UTC",
    "updated_at": "2026-03-02T14:30:00Z"
  }
}
```

### Error Responses

<ResponseField name="404 Not Found">
  Website does not exist
</ResponseField>

<ResponseField name="400 Bad Request">
  Invalid domain format (same validation as create endpoint)
</ResponseField>

## Delete a website

<Warning>
  This permanently deletes the website and **all associated analytics data** (events, sessions, goals). This action cannot be undone.
</Warning>

<CodeGroup>
```bash cURL
curl -X DELETE https://analytics.example.com/api/websites/site_a1b2c3d4e5 \
  -H "Authorization: Bearer spk_selfhosted_abc123..."
```
</CodeGroup>

### Path Parameters

<ParamField path="id" type="string" required>
  Website ID (format: `site_*`)
</ParamField>

### Response (204 No Content)

Empty response body on successful deletion.

### Error Responses

<ResponseField name="404 Not Found">
  ```json
  {
    "error": {
      "code": "not_found",
      "message": "Website not found"
    }
  }
  ```
</ResponseField>

## Website ID Format

All website IDs follow the format:

```
site_<10 random alphanumeric chars>
```

Example: `site_a1b2c3d4e5`, `site_x9y8z7w6v5`

This ID is used:
- In API endpoints as the `:id` path parameter
- In the tracking snippet as `data-website-id`
- In query endpoints to scope analytics data
